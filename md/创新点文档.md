# 项目创新点文档

**项目**：literature-ai 科研文献智能解析与知识服务系统

本文件梳理本系统在既有「单篇论文 RAG」基础上的核心创新点，用于答辩 / 结题 / 比赛陈述。

---

## 总述：从「读一篇」到「用一批」的能力跃迁

多数同类系统停留在「上传单篇 PDF → 问答」的层面。本系统在此之上，构建了
**「解析 → 检索增强 → 跨论文知识关联 → 主动推荐」** 的完整科研知识服务闭环，
形成四个可讲、可演示、可量化的创新点。

---

## 创新点一：跨论文对比与综述生成（Multi-Paper Comparison）

### 创新性
突破「单篇孤立分析」的局限，让系统能对**多篇论文做结构化横向对比**，并自动生成综述评述——这是科研工作者（尤其是做文献综述、选题调研时）的高频刚需，多数轻量工具不具备。

### 技术要点
- **结构化对齐**：复用 LLM 抽取的统一字段（研究问题 / 方法 / 数据集 / 指标 / 创新点 / 局限），把异构论文拉齐到同一比较框架。
- **多视角对比**：支持「综合 / 方法 / 实验」三种视角切换，同一套数据按需重组，满足不同调研目的。
- **对比 + 生成双层输出**：上层是可核对的结构化表格（来自数据库，秒级呈现），下层是 LLM 生成的综述评述（语义层面的横向分析）。

### 体验创新
表格数据与 LLM 生成解耦：**表格即时呈现、综述流式输出**，用户几乎无等待，规避了长文本生成的「干等」痛点。

---

## 创新点二：混合检索 + 可扩展重排（Hybrid Retrieval）

### 创新性
将检索质量本身作为工程创新点。多数 RAG 系统只用单一向量检索，对**专有名词、术语、缩写**（如模型名、数据集名）召回不稳。本系统采用**关键词 + 语义双通道混合检索**，显著改善此类查询。

### 技术要点
- **BM25 + 向量双通道**：语义向量（bge-large-zh）负责「意思相近」，BM25 负责「关键词精确命中」，互补。
- **RRF 融合**：用 Reciprocal Rank Fusion 融合两路排名，无需调权、鲁棒性好。
- **可扩展重排**：架构预留 cross-encoder 重排接口（开关式），可按算力渐进增强。

### 可量化
提供「混合开 / 关」开关，可在测试问答集上做**消融对比**，用命中率 / 相关度提升作为量化论据——这是答辩里有说服力的「实验」。

---

## 创新点三：知识图谱可视化 + AI/规则协同的可信构建（Knowledge Graph）

### 创新性
把用户文献库从「列表」升级为「**关系网络**」：论文间的引用脉络、共用数据集 / 方法一目了然，帮助用户快速把握一个方向的技术谱系。

### 技术要点（核心亮点：可信构建）
知识图谱的最大难点是「**自动构建易出错**」。本系统提出 **「AI 负责语义、规则负责可靠、用户负责校正」** 的三层协同：
1. **AI 语义层**：LLM 负责实体抽取、别名归一、模糊引用判定（识别"BERT"与"预训练语言模型"等语义等价）。
2. **规则兜底层**：置信度阈值过滤——**宁可漏连，不可错连**；匹配不上不造节点。
3. **人工校正层**：每条边可溯源（查看构建依据），支持手动删除 / 编辑。

> 设计哲学：**不追求 100% 正确，而追求「错了不误导 + 可溯源 + 可纠错」**——这是面对 LLM 幻觉的成熟工程应对，本身即为创新设计点。

### 体验
基于 vis-network 的可交互图谱：拖拽、缩放、点击节点跳详情、点击边看来源。

## 创新点四：管理员运营 + 标签匹配的双轨推荐（Hybrid Recommendation）

### 创新性
首页「精选推荐」采用**运营位 + 轻量算法**双轨制：
- **管理员运营位**：人工精选高质量论文推荐，保证首页内容质量与可控性。
- **标签匹配推荐**：基于 `field` / `tags` 重合度 + 未读优先，为用户做内容相关排序，实现「千人千面」的雏形。

### 技术要点
- 零额外模型成本的内容相关排序（标签交集加权 + 阅读状态）。
- 可平滑演进：预留升级到「向量相似度推荐」的空间（论文向量由 chunks 向量聚合而来）。

## 创新点小结表

| 创新点 | 关键词 | 差异化价值 |
|--------|--------|-----------|
| 跨论文对比 | 结构化对齐 · 三视角 · 表格+综述 | 从「读一篇」到「比一批」 |
| 混合检索 | BM25+向量 · RRF · 可量化消融 | 检索质量工程化、可实验验证 |
| 知识图谱 | AI+规则+人工三层可信构建 | 直面 LLM 幻觉的成熟工程设计 |
| 双轨推荐 | 运营位 + 标签匹配 | 内容可控 + 个性化雏形 |

---

## 贯穿全局的工程亮点（加分项）
- **流式体验**：QA / 报告 / 对比综述均 SSE 流式，规避长等待。
- **解耦设计**：`search_chunks(chunks_data=...)` 显式传参，检索模块与数据库解耦，便于测试与复用。
- **可信可控**：面对 LLM 的不确定性，统一采用「AI 提质 + 规则兜底 + 人工可校正」的策略，贯穿图谱、引用匹配等模块。
